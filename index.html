Personas & Storyboarding

Observing user behaviors for interfaces

Go to Observations

About

The goal of this exercise is to understand and examine how eveyday users interact with a public interface in order to gain insight into the UX design process and learn how to better construct similar interfaces.

Process

This task involves a 4-step Process

1
Interface
2
Observations
3
Personas
4
Storyboard

The public interface I chose to examine was a self-checkout kiosk located at CVS. I first analyzed all the different aspects of the interface, then observed and interviewed a few users about the process of using the interface. Then, I constructed personas based off those user experiences and finally created a storyboard to visually show the whole process of using that interface.


Interface

An interface is chosen. All interactive elements of the system are examined and noted.
Focus is given to the purpose and use of the interface; what problem it is trying to solve.

Physical Interface

This interface has both a physical and digital component. Along with the digital terminal, the kiosk also includes an area to scan items and a payment terminal on the right. The purpose of this interface is to automate the process of customer checkout. The interface is designed to be easy to understand on the first use and quick to complete.

Observations

Real world users are observed using the interface. Then, they are interviewed based on thier experineces.
Emphasis is put on objective observations and the user's goals and difficulties while using the interface.

Key Observations

- Two out of the three users were able to use the interface without the need of outside assistance
- Most of the users took a similar amount of time (around 2 min 35 sec) to finish using the interface
- The help button on the interface seemed to do nothing when one of the users pressed it, and the user had to physically go and ask a worker for help
- All users were immediately able to understand how to use the interfaces' scanner.

Questions

  • 1. What was the first thing that your eyes were drawn to when you saw the interface?
  • 2. Are there any features of a checkout with a store clerk that you would want the self-service kiosk to include?
  • 3. Did it take longer, shorter or the the expected amount of time to use?
  • 4. Was there anything about the interface which caused you great frustration?
  • 5. Were the phsyical and digital aspects of the interface well-integrated?

Responses

  • All of the users found the diagrams on the digital interface to be very helpful in understanding how to use the physical aspects of the interface
  • One of the users had coupons and was frustrated by the fact that when coupons are submitted, the reduction in cost is not calculated immediately but only when the user presses the pay button.
  • All the users felt the process of using the interface was very quick.
  • Another user voiced concern over the fact that the interface did not have a search bar, if the barcode does not work then there is no way to enter the item into the system.

Storyboard

One of those personas is placed into a storyboard. The storyboard conveys the persona's journey in using the chosen interface.
It offers an authentic scenario of what the persona might do in such a situation (and by proxy how certain users might act).

For the storyboard I chose the Diana persona.
The scenario of the storyboard is that Diana has some items she wants to pick up but is tight on time because she has a meeting she needs to attend in 45 minutes.
